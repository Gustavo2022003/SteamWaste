Troubleshooting

This page provides solutions to common issues encountered in this application. If you've experienced any issues not listed here, please report them so we can address and fix them or visit the issues page at our GitHub.

⚠️ IMPORTANT

Due to constraints imposed by the Steam API — including the possibility of temporarily blocking the host IP if too many requests are made in a short period — this website may display inaccurate or outdated information, which could adversely affect the user experience. We therefore strongly recommend running this application locally on your computer. Please refer to this tutorial for a detailed step-by-step guide.

"My information is not being updated on the website."

This may happen due to our information caching system. To avoid overloading the Steam API with too many requests per SteamID, we limit these requests to being made only once every 24 hours. That means, if it's your first request of the day through our site, the information will be stored in cache for later reuse. This significantly reduces the response time for subsequent queries within that 24-hour period. Once that period expires, the next request may take longer, just like the first one, as we need to update and replace the data, then store it in cache again.

"The price displayed does not match the price on the Steam product page"

This may occur occasionally. It's important to note that we use the latest price provided by the API, and in some cases, this may be the full price (without any discounts). The API may take a few days to update the price in sync with the value shown on the Steam product page.

"Internal Server Error - 500"

If you encounter this issue, please report it here immediately. In the issue description, kindly provide details on how you found the error (if possible) and on which page (E.g. "/example") it occurred. Provide the most information as you can.

"The website is displaying information that does not match my Steam account"

This may occur due to rate limit issues with the API. To avoid this, we recommend running the application locally. If the issue persists, please report it here.
